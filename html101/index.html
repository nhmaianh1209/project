Bước 1

3 lớp xây dựng 1 landing page

Trước khi bắt tay vào làm, bạn cần phân biệt rõ 3 lớp này — vì cả hành trình phía sau chỉ là đi từ lớp này sang lớp kia.

💭

Lớp 1 — Nội dung

  • Là "cái muốn nói" — thông tin, câu chữ, hình ảnh bạn muốn truyền tải
  • Có thể tự viết sẵn, hoặc nhờ AI (Claude/ChatGPT) hỗ trợ xây dựng từ đầu
  • Ở lớp này, nội dung chưa gắn với định dạng trình bày nào cả
📄

Lớp 2 — Hình thức

  • Cùng 1 nội dung, có thể trình bày bằng nhiều định dạng: Word, PPT, PDF, hoặc HTML
  • 4 định dạng đều "chứa" được nội dung, nhưng chỉ HTML là định dạng trình duyệt hiểu và hiển thị trực tiếp được
→ Hướng dẫn chi tiết ở Bước 2
🌐

Lớp 3 — Mang lên online

  • Dù chọn hình thức nào, file đó vẫn đang "offline" — giống như 1 file Word: lưu được, gửi qua Zalo/email được, nhưng người nhận phải tải về mới mở xem
  • Riêng file HTML, ta có thể "mang lên online" — gắn tên miền, có link để chia sẻ — điều Word/PPT/PDF không tự làm được

Bước 2.2

Bắt đầu cuộc trò chuyện

Bạn đã quen với việc viết prompt rồi, nên phần này mình tập trung vào điều quan trọng nhất: đảm bảo prompt cho ra đúng loại output cần thiết để deploy dễ dàng ở Bước 3.

Mô tả rõ: mục đích trang, nội dung cần có, phong cách mong muốn, và yêu cầu output ở dạng 1 file HTML/interactive landing page HTML duy nhất, gồm luôn CSS và JavaScript trong cùng file.

Prompt mẫu — copy và chỉnh lại theo ý bạn

Prompt mẫu #1 — Trang giới thiệu bản thân Hãy tạo cho tôi 1 trang landing page giới thiệu bản thân, gồm: ảnh đại diện, phần giới thiệu ngắn (2-3 câu), và danh sách liên kết mạng xã hội. Phong cách tối giản, tông màu xanh navy và cam ấm.
Quan trọng: xuất ra 1 file HTML duy nhất, gồm luôn CSS và JavaScript trong cùng file (không tách file riêng).
Prompt mẫu #2 — Trang sự kiện / workshop Tạo giúp tôi 1 landing page quảng bá 1 buổi workshop, gồm: tiêu đề, thời gian - địa điểm, 3 lý do nên tham gia, và 1 nút đăng ký. Phong cách chuyên nghiệp, gọn gàng.
Quan trọng: xuất ra 1 file HTML duy nhất, gồm luôn CSS và JavaScript trong cùng file.

CSS giúp trang có màu sắc/bố cục đẹp, JavaScript giúp trang có thể tương tác (bấm nút, hiệu ứng). Yêu cầu gộp cả 2 vào chung 1 file để ở Bước 3 bạn chỉ cần upload đúng 1 file là chạy được toàn bộ.

✅ Checkpoint — thử luôn

Bước 2.4

Phản hồi (vòng lặp với Bước 3)

Chưa ưng ý chỗ nào — muốn đổi màu, thêm phần, đổi bố cục — chỉ cần nói đúng phần cần sửa. Không cần yêu cầu Claude làm lại từ đầu.

Claude sẽ cập nhật lại Artifact, bạn xem lại ở Bước 3, rồi tiếp tục phản hồi nếu cần. Lặp lại Bước 3 ⇄ Bước 4 cho đến khi ưng ý.

Bước 2.5

Tải file về

Bước 3

Đưa file HTML lên online — cần 1 tên miền, ai cung cấp?

File HTML ở Bước 2 vẫn đang "offline". Muốn có link để chia sẻ, bạn cần 1 nơi vừa lưu trữ file, vừa cấp cho bạn 1 tên miền để ai cũng truy cập được.

💡 Mẹo nhỏ

Bạn hoàn toàn có thể nhờ chính Claude hướng dẫn từng bước cụ thể để đưa file lên Netlify hoặc GitHub — chỉ cần gõ ví dụ: "Hướng dẫn tôi từng bước để đưa file HTML này lên Netlify". Nếu gặp khó khăn ở bước nào, chụp màn hình màn hình đang gặp lỗi và gửi trực tiếp cho Claude — Claude sẽ xem ảnh và chỉ dẫn chính xác hơn rất nhiều so với việc chỉ mô tả bằng lời.

🚀 Xem hướng dẫn Netlify
🐙 Xem hướng dẫn GitHub Pages
1

Tạo tài khoản Netlify

Vào app.netlify.com/signup, đăng ký bằng email hoặc bằng tài khoản Google/GitHub có sẵn. Miễn phí, không cần nhập thẻ ngân hàng.

2

Vào Netlify Drop

Truy cập app.netlify.com/drop, kéo-thả file HTML của bạn vào đúng khung ở giữa trang.

3

Nhận link ngay lập tức

Ngay sau khi thả file, Netlify tạo ngay 1 link public dạng tên ngẫu nhiên (vd: chipper-cat-123.netlify.app) — click vào để mở thử.

4

(Tuỳ chọn) Đổi tên subdomain

Vào Site settings → Change site name, đổi phần subdomain thành tên bạn muốn, ví dụ mai-anh-portfolio.netlify.app.

5

(Tuỳ chọn) Gắn tên miền riêng

Vào Domain settings → Add custom domain nếu sau này muốn có 1 domain hoàn toàn riêng (không còn ".netlify.app").

GitHub có khái niệm repository (repo) — hiểu đơn giản là "1 thư mục online" chứa toàn bộ file của 1 dự án.

1

Tạo tài khoản GitHub

Vào github.com/join, đăng ký bằng email. Miễn phí.

2

Tạo 1 repository mới

Vào github.com/new, đặt tên cho repo (nếu muốn dùng làm trang cá nhân chính, đặt tên đúng dạng username.github.io) → bấm Create repository.

3

Upload file trực tiếp qua giao diện web

Trong repo vừa tạo, bấm Add file → Upload files, kéo-thả file HTML vào, cuộn xuống bấm Commit changes. Không cần cài đặt gì, không cần dùng dòng lệnh.

4

Bật GitHub Pages

Vào tab Settings của repo → mục Pages ở thanh bên trái → chọn Branch main và thư mục / (root) → bấm Save để bật GitHub Pages.

5

Lấy link và mở thử

Chờ khoảng 1-2 phút, link sẽ hiện ngay trong chính mục Settings → Pages, dạng username.github.io/ten-repo.

💡 Ghi chú nhỏ để không bỡ ngỡ nếu học tiếp: Git là công cụ theo dõi thay đổi file, và commit là "1 lần lưu" thay đổi đó lại. Ở mức này bạn chưa cần dùng đến, chỉ cần biết tên.

Hoàn tất

🎉 Landing page của bạn đã online!

Giờ đây bất kỳ ai có link cũng có thể mở xem trang của bạn — không cần gửi file, không cần cài gì cả.

Việc nên làm ngay

  • Lưu lại link vào 1 nơi dễ tìm
  • Thử mở link bằng điện thoại
  • Gửi thử cho 1 người bạn để họ mở xem

Những điểm có thể cải tiến hơn

  • Với landing page cần update nhiều lần, thay vì chỉ tạo trong 1 cuộc trò chuyện, có thể tạo trong 1 Project để giữ được bối cảnh xuyên suốt
  • Tìm hiểu thêm về HTML, frontend, backend để ứng dụng được nhiều tình huống hơn

Bạn đã đi từ 1 ý tưởng, đến file HTML, đến 1 landing page thật sự có link — hoàn toàn không cần biết code. Bất cứ khi nào quên bước nào, quay lại xem hoặc mở bảng thuật ngữ ở góc phải bên dưới.
